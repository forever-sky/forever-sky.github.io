路由交换ospf协议

发表于 2019-12-26 分类于 路由交换
本文字数： 440

##1、 Hello报文的作用是什么？
发现及维持邻居关系，选举DR、BDR。

##2、路由器间的邻接关系是怎样建立的？
路由器首先发送拥有自身ID信息（Loopback端口或最大的IP地址）的Hello报文。与之相邻的路由器如果收到这个Hello报文，就将这个报文内的ID信息加入到自己的Hello报文内。如果路由器的某端口收到从其他路由器发送的含有自身ID信息的Hello报文，则它根据该端口所在网络类型确定是否可以建立邻接关系。

##3、指定路由器(DR)、备份指定路由器(BDR)是怎样选举出来的？
选举利用Hello报文内的ID和优先权(Priority)字段值来确定。先看优先级，再看rid 谁大听谁的。

##4、 Database Description报文的作用是什么？
描述本地LSDB的情况。

##5、Link State Request报文的作用是什么？
向对端请求本端没有或对端更新的LSA。

##6、Link State Acknowledge报文的作用是什么？
收到LSU报文后，进行确认。

ubuntu 16.04 安装NFS服务记录

发表于 2019-12-23 更新于 2019-12-26 分类于 服务配置
本文字数： 613

sudo apt install nfs-kernel-server

sudo vi /etc/exports (一般修改NFS配置文档后，是不需要重启NFS的，直接在命令执行/etc/init.d/nfs reload或exportfs –rv即可使修改的/etc/exports生效。)

/home/share 目录可读写，并且不限制用户身份，共享给192.168.8.0/24网段的所有主机；但仅有该文件/目录的建立者与root能够删除自己的目录或文件。

mkdir -p /home/share
chmod 1777 /home/share/
sudo vi /etc/exports
/home/share 192.168.8.0/24 (rw,no_root_squash)

/home/zhidata 这个目录仅共享给192.168.8.7这台主机，以供该主机上面的wen这个用户来使用（也就是说wen在NFS服务器端和客户端上均有账号，且账号均为wen）；

mkdir -v /home/zhidata
useradd wen
passwd wen
sudo vi /etc/exports
/home/zhidata 192.168.8.7 (rw)

##1、RPC服务重新启动后，为何要重新启动NFS服务？
NFS SERVER就无法向RPC服务区注册,如果RPC服务重新启动，原来已经注册好的NFS端口数据就会全部丢失。因此此时RPC服务管理的NFS程序也要重新启动以重新向RPC注册。
